# BBCode

BBCode - это синтаксис разметки, используемый на osu! форумах, и в подавляющем большинстве других форумов в интернете, для того, чтобы включить форматирование текста. Он состоит из тегов, которые предоставляют более удобные возможности по форматированию сообщений. В osu! форумах, BBCode используется в постах, подписях, страницах пользователей и описаниях карт.

## Примечание

Хоть редактор сообщений  и предоставляет несколько основных инструментов форматирования, также можно и вручную вписать BBCode. Теги BBCode не зависят от регистра символов.

### Модель поведения кнопки BBCode

Если текст не выделен, то нажатие кнопки окружит теги вокруг расположения текстового курсора в редакторе записей. Когда текст выделен в редакторе записей, то нажатие кнопки окружит теги вокруг выделенного текста.

### Комбинирование тегов

Теги можно комбинировать для улучшения форматирования текста. Порядок размещения тегов **должен** соблюдаться при их комбинировании. Это важно. Несоблюдение порядка / вложения приведут к некорректному форматированию текста / поста.

Например (обратите внимание на порядок):

- `[centre]` `[b]` *text* `[/b]` `[/centre]` корректный, но
- `[b]` `[centre]` *text* `[/b]` `[/centre]` некорректный.

## Список тегов BBCode

### Жирный шрифт

**Выделяет** слова или весь абзац , **более жёстче** нежели [курсивом](#Курсив).

- Соответствующая кнопка: ![bold img](img/bold.png)
- Примечание:
  - Используйте в меру. Чрезмерное использование может вызвать дискомфорт и затруднения при чтении.

- Конструкция:

  ```
  [b] ... [/b]
  ```

### Курсив

**Выделяет** слова или обзац, *более мягче* нежели [жирными шрифтом](#bold).

- Соответствующая кнопка: ![Italic button](img/italic.png)
- Примечание: 
  - Используйте в меру. Чрезмерное использование может сказаться на акцентировании.

- Конструкция: 

  ```
  [i] ... [/i]
  ```

### Подчеркивание

**Рисует горизонтальную линию** под словом(ами) или абзацем.

- Соответствующая кнопка: N/A
- Примечание: 
  - Используйте в меру. Чрезмерное использование может вызвать дискомфорт и затруднения при чтении.

- Конструкция: 

  ```
  [u] ... [/u]
  ```

### Зачёркивание

*Так же известно как **перечёркинвание**.*

**Рисует горизонтальную линию** через середину слова или целого абзаца.

- Соответствующая кнопка: ![Strike button](img/strike.png)
- Примечание: 
   - Используйте в меру. Чрезмерное использование может вызвать дискомфорт и затруднения при чтении.

- Конструкция: 

  ```
  [strike] ... [/strike]
  ```

### Цвет

**Добавляет цвет** к некоторому тексту.

- Соответствующая кнопка: N/A
- Примечание: 
   - Цветовая палитра расположена в правой части текстового поля. 
   - Если вы выбираете цвет, убедитесь, что он контрастен с фоном, чтобы любой мог его прочитать.
   - Аргумент `HEXCODE` не должен содержать кавычек.

- Конструкция:

  - `HEXCODE` - цвет в шестнадцатеричном формате или название цвета.
    - Такой формат должен начинаться с `#` , а затем 6 последовательных символов (0 - 9, A - F).
    - Название цвета должно быть в HTML.

  
```
  [color=HEXCODE] ... [/color]
  ```

Список [цветов](https://en.wikipedia.org/wiki/X11_color_names#Color_name_chart) в разделе "Color name".

### Размер шрифта

Относительно регулирует размер текста.

- Соответствующая кнопка: ![Font size options](img/font-size.png)
- Примечание: 
  
  - Используйте тег в меру.
  - Доступно только 4 размера:
    - `50` (мелкий)
    - `85` (маленький)
    - `100` (нормальный, по умолчанию)
    - `150` (большой)
  - Если вы введете значение, которое не является одним из перечисленных выше, он будет использовать размер шрифта по умолчанию.
  - Аргумент `NUMBER` не должен содержать кавычек.

 ```
  [size=NUMBER] ... [/size]
  ```
### Спойлер

*Не путать со [спойлербоксом](#спойлербокс).*

**Покрывает текст** чёрным фоном. А точнее скрывает его.

Это может быть полезно, если Вы говорите о критической сцене в телешоу, сериале, фильме или аниме, но не хотите рассказывать об этом другим. Либо просто хотите скрыть какую-либо информацию. Однако читатель все равно может прочитать текст, выделив его.

- Соответствующая кнопка: N/A
- Примечание: 
  - Если Вы используйте спойлер с [цветовым](#цвет) тегом, то цвет останется на чёрном фоне.
 
- Конструкция: 

 ```
  [spoiler] ... [/spoiler]
 ```
### Спойлербокс

*Не путать с [блоком](#блок) и [спойлером](#спойлер).*

**Скрывает абзац** внутри предварительно обозначенного поля от читателя. Однако читатель все еще может прочитать абзац, нажав на спойлербокс, чтобы развернуть скрытый абзац.

- Соответствующая кнопка: ![Spoilerbox button](img/spoilerbox.png)
- Примечание: 
   - Создаёт [блок](#блок) с заголовком `свернутого текста`.

- Конструкция: 

 ```
  [spoilerbox]
  ...
  [/spoilerbox]
  ```

### Блок

*Не путать со [сбойлербоксом](#спойлербокс).*

**Скрывает абзац** внутри именуемого поля от читателя. Однако читатель все еще может прочитать абзац, нажав на заголовок, чтобы развернуть скрытый абзац.

Они обычно используются, чтобы скрыть большие изображения, или большое количество изображений.

- Соответствующая кнопка: N/A
- Примечание: 
  - По умолчанию создает блок без заголовка.
    - Если Вы оставите аргумент `NAME` пустым (или не будет содержать его), высота блока будет очень мала!
  - Аргумент `NAME` не должен содержать кавычки.
  - Аргумент `NAME` может содержать проблемы.
 
 - Конструкция: 
  - `NAME` -заголовок блока.

 ```
  [box=NAME]
  ...
  [/box]
  ```

### Цитата

Цитирует слово / абзац.

- Соответствующая кнопка: N/A
- Примечание: 
  - Приятно отметить автора цитаты, если Вы кого-то цитируете, используйте аргумент `NAME`.
  - Аргумент `NAME` должен содержать кавычки.
  - Существует кнопка цитирования для каждого поста (нижний правый угол), которая автоматически цитирует текст пользователя.
    - Если Вы сделали это, то пост (когда Вы нажмете `Post`)будет размещено в текущей теме.

- Конструкция: 
  - `NAME` - заголовок блока (должно быть заключено в кавычки).

 ```
  [quote=NAME]
  ...
  [/quote]
  ```

### Код

Отформатируйте текст с помощью моноширинного семейства шрифтов и поместите его в серую рамку. Это полезно, когда Вы размещаете код для раскадровки или исходный код из проекта.

- Соответствующая кнопка: N/A
- Примечание: 
   - Позволит сохранить белый интервал, а это значит, что ни одна из линий не прервется, разве что, если Вы не нарушите ее.
   - Переполнение текстовых строк приведет к появлению горизонтального бегунка.

- Конструкция: 

 ```
  [code]
  ...
  [/code]
  ```

### URL-адрес

Добавляет ссылку.

Вам не нужно использовать этот тег, если Вы не хотите использовать текстовые ссылки. Osu! форумы будут автоматически связывать вставленные URL адреса внутри поста.

- Соответствующая кнопка: ![URL button](img/url.png)
- Примечание: 
  - Аргумент `LINK` не должен содержать кавычек.

- Конструкция: 
  - `LINK` - ссылка.
  - `TEXT` - текст ссылки.

 ```
  [url=LINK]TEXT[/url]
  ```

### Профиль

Связывает профиль пользователя, используя его имя.

Использование этой кнопки крайне не рекомендуется! Проблема с использованием этого тега заключается в том, что имена пользователей могут быть изменены после приобретения [osu!supporter](/wiki/osu!supporter). И как только они это сделают, связь c их профилем оборвется.

Вместо этого рекомендуется использовать [URL-адрес](#url-адрес) (используя ID пользователя).

- Соответствующая кнопка: N/A
- Примечание: 
   - Не рекомендуется для исопльзования!
   - Должен быть опеределен аругмент `USER`.

- Конструкция: 
   - `USER` - имя пользователя.

 ```
  [profile]USER[/profile]
  ```

### Список

Форматирует текст в список.

В BBCode есть две части для составления списка: оболочка и маркировка. Где маркировка помещается внутри оболочки.

- Соответствующая кнопка: N/A
  - Маркированный список:![List button](img/list.png)
  - Нумерованный список: ![Numbered list button](img/list-numbered.png)
  - Список пуля: N/A
  - Тип списка: N/A

- Примечание:
  - Все маркировки должны содержать теги list/list.
  - Допустимые теги для аргумента `TYPE`:
    - *(Пустой)* - маркированный
    - `1` - нумированный
    - `a` - алфавитный (строчные)
    - `A` - алфавитный (заглавные)
    - `i` - римские цифры (строчные)
    - `I` - римские цифры (заглавные)
  - По умолчанию, маркерованный список состоит из квадратных точек.
  - Списки внутри списков, как известно, ломаются.

- Конструкция: 
  
  - `TYPE` - один из приведенных выше видов списков.
  - Если не указан `TYPE`, то будут использоваться квадратные точки.

 ```
  [list=TYPE]
  [*]...
  [/list]
  ```

### Изображение

**Отображает изображение из онлайн источника, откуда оно было взято**.

Изображение будет отображаться пока действителен URL-адрес .

**Не указывайте ссылку на изображение непосредственно из локального пути к файлу!** Использование `C:\Users\Name\Pictures\image.jpg` **не будет работать**.

Пожалуйста, загружайте изображение на зарекомендованные сайты обмена изображениями, такие как [imgur](https://imgur.com) или [puush](https://puush.me). Как только изображение будет успешно загружено, скопируйте и вставьте прямую ссылку, предоставленную между тегами. Также обратите внимание, что некоторые веб-сайты недолюбливуют прямые ссылки  (иначе называемые *горячими ссылками*). С сайтами перечисленными выше должно быть всё в порядке.

- Соответствующая кнопка: N/A
- Примечание: 
  - Если у Вас много изображений или они слишком большие, то рекомендуем разместить внутри блока.

- Конструкция: 
  - `LINK` - прямая ссылка на изображение.

 ```
  [img]LINK[/img]
  ```

### YouTube

Прикрепляет видео с YouTube в Ваш пост.

- Соответствующая кнопка: N/A
- Примечание: 
  - Если у Вас много изображений или они слишком большие, то рекомендуем разместить внутри блока.

- Конструкция: 
  - `VIDEO_ID` идентификатор видео на YouTube (длиной 11 символов), **не** весь URL-адрес.

 ```
  [youtube]VIDEO_ID[/youtube]
  ```

### Аудио

**Показывает аудиоплеер HTML5 из онлайн источника**.

Аудиофайл будет работать пока действителен URL-адрес.

**Не указывайте ссылку на аудиофайл непосредственно из локального пути к файлу!** Использование `C:\Users\Name\Music\audio.mp3` **не будет работать**.

Пожалуйста, загружайте аудиофайлы на зарекомендованные сайты, такие как [puush](https://puush.me). Как только аудиофайл будет успешно загружено, скопируйте и вставьте прямую ссылку, предоставленную между тегами. Также обратите внимание, что некоторые веб-сайты недолюбливуют прямые ссылки (иначе называемые *горячими ссылками*). Пожалуйста, имейте в виду, что на некоторых сервисах по обмену файлами существуют проблемы с пиратством. С сайтами перечисленными выше должно быть всё в порядке.

- Соответствующая кнопка: N/A
- Конструкция: 
  - `URL` - ссылка на аудиофайл.

 ```
  [audio]URL[/audio]
  ```

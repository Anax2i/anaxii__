# Modding

**Modding** est l'acte de fournir un retour d'information sur les [beatmap](/wiki/Beatmaps). Un utilisateur qui modifie est appelé *modder*, et ses messages collectifs sur une discussion de beatmap sont appelés *mod*.

Les mods sont principalement utilisés pour améliorer les beatmaps visant à [les ranked](/wiki/Beatmaps#ranked). Parce qu'il s'agit d'un processus détaillé et analytique, la modification des beatmaps est également un moyen populaire pour les utilisateurs d'améliorer leur compréhension des beatmaps.

## Trouver des beatmaps à modifier

Bien qu'il existe d'innombrables façons de découvrir les beatmaps à moduler, voici quelques méthodes courantes :

- Le [Forum Modding Queues](https://osu.ppy.sh/community/forums/60) est la plateforme la plus activement utilisée pour trouver des beatmaps à moduler et demander des mods. Un utilisateur peut créer son propre fil de discussion, appelé *queue*, pour que d'autres utilisateurs puissent poster des beatmaps qu'ils souhaitent modifier.

  Le créateur de chaque file d'attente dicte les règles de la file d'attente, en contrôlant les beatmaps qui sont viables pour recevoir les mods. Les files d'attente sont généralement ouvertes pour les demandes de deux types de mods :

  - **Mod normal (NM):** Un mod qui sera fourni sans travail de la part du beatmappers demandeur.
  - **Mod for mod (M4M):** Un échange qui exige que chaque utilisateur modifie le beatmap de l'autre.

- Les moddeurs connus notent souvent sur leur profil d'utilisateur s'ils acceptent ou non les demandes de mod.

- Il n'est **pas** recommandé de chercher dans la liste des beatmaps en attente pour trouver des beatmaps à moduler car la plupart des soumissions ne visent pas le statut de Ranked, donc leurs créateurs ne répondront potentiellement pas aux mods.

## Comment mod

Pour modifier un beatmapset, commencez par télécharger la version la plus récente et ouvrez la [page de discussion](/wiki/Beatmap_Discussion). Jouez la [difficulté](/wiki/Difficulties) vous voulez faire des modifications et garder à l'esprit les questions liées au jeu. Une fois terminé, ouvrez la difficulté dans l'éditeur et analysez-la pour d'autres problèmes.

Lorsque vous rencontrez un problème, sélectionnez les objets pertinents, puis utilisez le raccourci de copie (`Ctrl` + `C`) pour copier leurs informations d'horodatage sous forme de texte. Sur la discussion de la carte des temps, collez (`Ctrl` + `V`) les éventuels horodatages applicables et vos commentaires. Le site `Chronologie` est utilisé pour les questions concernant des parties spécifiques d'une carte de rythme, l'onglet `Général (Cette difficulté)` est utilisé pour les commentaires sur des sections plus importantes d'une carte de rythme, et l'onglet `Général (Toutes les difficultés)` est utilisé pour les questions applicables à toutes les difficultés sur un beatmapset.

Tous les messages de mod doivent aborder la nature et la raison du problème, ainsi que la manière de le résoudre. Voir le [Code de conduite pour la modification et la cartographie](/wiki/Rules/Code_of_Conduct_for_Modding_and_Mapping#making-a-mod-post) pour plus de détails sur la façon de formater les messages de mod.

Les Beatmappers fournissent régulièrement [kudosu](/wiki/Glossary#kudosu!) pour modérer les messages qu'ils jugent utiles en cliquant sur le bouton de vote *thumbs up*. Cette statistique est suivie sur le profil d'un individu et est référencée lors de la candidature pour rejoindre le [Beatmap Nominators](/wiki/People/The_Team/Beatmap_Nominators).

## Types de postes de mod

Le but de chaque `Problème` ou `Suggestion` posté sur une page de discussion de beatmap est d'expliquer tout ce qui concerne un beatmap dans le contexte de sa conception.

De nombreux messages de mod traitent des erreurs dans un beatmap. Par exemple, un clap hitsound manquant qui serait attendu par le modèle de frappe de la beatmap est probablement une erreur.

Un autre type courant de mod post est de fournir des moyens alternatifs de beatmapping. Bien qu'une partie d'un beatmap puisse être acceptable dans sa forme actuelle, un moddeur peut expliquer comment l'améliorer davantage en le concevant différemment.

Dans les beatmaps visant à obtenir le statut de Ranked, en s'assurant qu'ils respectent la [Ranking Critera](/wiki/Ranking_Criteria) est un must. Un certain nombre de postes de mod traitent de questions citées directement à partir de ces critères.
